[Overview]
== Overview

CustomerCare microservice belongs to KYC-Domain. The CustomerCare is responsible to provide all operations related to Customer.
The entire microservice provides CRUD operations exposed through RESTFul APIs as below examples.



== POST


This method creates a brand-new customer. The operation inits by validating all required customer
fields, in case of success the operation consults KYC APIs in order to complete
the operation. A business error is being returned if the customer does not fulfil the criteria.

This API operation returns the newly created customer's id in header.
